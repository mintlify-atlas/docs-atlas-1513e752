---
title: Register
api: POST /auth/register
---

Registers a new user account. After successful registration, a verification email is sent to the provided email address.

## Request Body

<ParamField body="name" type="string" required>
  User's first name. Must contain at least one letter (A-Z, a-z, or accented characters)
</ParamField>

<ParamField body="lastname" type="string" required>
  User's last name
</ParamField>

<ParamField body="email" type="string" required>
  User's email address. Must be unique
</ParamField>

<ParamField body="password" type="string" required>
  User's password
</ParamField>

## Response

<ResponseField name="message" type="string">
  Success message indicating registration was successful and verification email was sent
</ResponseField>

<ResponseField name="user" type="object">
  Created user information
  
  <ResponseField name="user.id" type="integer">
    Unique user identifier
  </ResponseField>
  
  <ResponseField name="user.name" type="string">
    User's first name
  </ResponseField>
  
  <ResponseField name="user.lastname" type="string">
    User's last name
  </ResponseField>
  
  <ResponseField name="user.email" type="string">
    User's email address
  </ResponseField>
  
  <ResponseField name="user.role" type="string">
    User's role (default: "USER")
  </ResponseField>
</ResponseField>

## Example Request

```bash
curl -X POST https://api.example.com/auth/register \
  -H "Content-Type: application/json" \
  -d '{
    "name": "John",
    "lastname": "Doe",
    "email": "john.doe@example.com",
    "password": "securePassword123"
  }'
```

## Example Response

```json
{
  "message": "Fuiste registrado satisfactoriamente, revisa tu casilla de correo para ingresar luego el token para activar la cuenta",
  "user": {
    "id": 123,
    "name": "John",
    "lastname": "Doe",
    "email": "john.doe@example.com",
    "role": "USER"
  }
}
```

## Error Responses

<ResponseField name="400 Bad Request">
  Returned when required fields are missing or name doesn't contain at least one letter
  
  ```json
  {
    "error": "El nombre debe contener al menos una letra"
  }
  ```
  
  Or:
  
  ```json
  "Datos incompletos para el registro"
  ```
</ResponseField>

<ResponseField name="409 Conflict">
  Returned when the email is already registered and the account is active
  
  ```json
  "La cuenta ya está activa. Inicia sesión."
  ```
</ResponseField>

<ResponseField name="200 OK (Already Registered)">
  If the email exists but the account is not yet verified, a new verification token is sent
  
  ```json
  "La cuenta ya existe pero no está activa. Se ha reenviado el token de verificación a tu correo."
  ```
</ResponseField>
