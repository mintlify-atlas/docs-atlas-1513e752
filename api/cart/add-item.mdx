---
title: "Create Cart with Items"
api: "POST /cart"
description: "Create a new shopping cart with items and generate a purchase order"
---

## Authentication

This endpoint requires a valid JWT token in the Authorization header. The user information is extracted from the token.

## Headers

<ParamField header="Authorization" type="string" required>
  Bearer token for authentication
  
  Example: `Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...`
</ParamField>

## Request Body

<ParamField body="items" type="array" required>
  Array of items to add to the cart
  
  <ParamField body="product" type="object" required>
    Product information
    
    <ParamField body="id" type="integer" required>
      ID of the product to add
    </ParamField>
  </ParamField>
  
  <ParamField body="quantity" type="integer" required>
    Quantity of the product to add (must not exceed available stock)
  </ParamField>
</ParamField>

## Response

Returns a Purchase object with status PENDING.

<ResponseField name="id" type="integer">
  Unique identifier of the purchase
</ResponseField>

<ResponseField name="status" type="string">
  Purchase status (PENDING, CONFIRMED, or CANCELLED)
</ResponseField>

<ResponseField name="date" type="string">
  Purchase creation date and time (ISO 8601 format)
</ResponseField>

<ResponseField name="reservationTime" type="string">
  Time when the products were reserved (ISO 8601 format)
</ResponseField>

<ResponseField name="cart" type="object">
  The created cart
  
  <ResponseField name="id" type="integer">
    Cart ID
  </ResponseField>
  
  <ResponseField name="finalPrice" type="float">
    Total price of all items
  </ResponseField>
  
  <ResponseField name="items" type="array">
    Array of cart items with updated product stock
  </ResponseField>
</ResponseField>

## Business Logic

1. **Stock Validation**: Before creating the cart, the system validates that:
   - All products exist
   - All products are active
   - Sufficient stock is available for each product

2. **Stock Deduction**: Upon successful cart creation, the product stock is automatically reduced by the requested quantities

3. **Purchase Creation**: A Purchase order is created with status PENDING and the current timestamp

## Error Responses

- **400 Bad Request**: Product not found, product is not active, or insufficient stock
- **401 Unauthorized**: User session is not active or token is invalid

### Error Headers

Error details are returned in the `X-Error-Message` header:
- "Producto no encontrado: ID {id}"
- "El producto '{title}' no está activo"
- "Stock insuficiente para el producto '{title}'. Disponible: {available}, Solicitado: {requested}"

```bash cURL
curl -X POST "https://api.example.com/cart" \
  -H "Authorization: Bearer YOUR_JWT_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "items": [
      {
        "product": {
          "id": 45
        },
        "quantity": 2
      },
      {
        "product": {
          "id": 78
        },
        "quantity": 1
      }
    ]
  }'
```

```javascript JavaScript
const response = await fetch('https://api.example.com/cart', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer YOUR_JWT_TOKEN',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    items: [
      {
        product: { id: 45 },
        quantity: 2
      },
      {
        product: { id: 78 },
        quantity: 1
      }
    ]
  })
});

const purchase = await response.json();
```

```python Python
import requests

headers = {
    'Authorization': 'Bearer YOUR_JWT_TOKEN',
    'Content-Type': 'application/json'
}

payload = {
    'items': [
        {
            'product': {'id': 45},
            'quantity': 2
        },
        {
            'product': {'id': 78},
            'quantity': 1
        }
    ]
}

response = requests.post('https://api.example.com/cart', 
                        headers=headers, 
                        json=payload)
purchase = response.json()
```
