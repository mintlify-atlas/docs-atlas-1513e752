## Overview

The Ecommerce Backend API uses MySQL as its primary database with Spring Data JPA and Hibernate ORM. The application connects to MySQL via JDBC and uses Hibernate for automatic schema management.

## Database Connection

### JDBC Configuration

The database connection is configured through environment variables that construct the JDBC URL:

```properties
spring.datasource.url=jdbc:mysql://${SPRING_DATASOURCE_URL}
spring.datasource.username=${SPRING_DATASOURCE_USERNAME}
spring.datasource.password=${SPRING_DATASOURCE_PASSWORD}
```

### Required Environment Variables

<ParamField path="SPRING_DATASOURCE_URL" type="string" required>
  MySQL server hostname, port, and database name.
  
  **Format:** `hostname:port/database_name`
  
  **Example:** `localhost:3306/ecommerce` or `mydb.us-east-1.rds.amazonaws.com:3306/ecommerce`
</ParamField>

<ParamField path="SPRING_DATASOURCE_USERNAME" type="string" required>
  Database user with sufficient privileges (SELECT, INSERT, UPDATE, DELETE, CREATE, ALTER, DROP).
</ParamField>

<ParamField path="SPRING_DATASOURCE_PASSWORD" type="string" required>
  Password for the database user.
</ParamField>

## JPA and Hibernate Configuration

### Schema Management

```properties
spring.jpa.hibernate.ddl-auto=update
```

<ParamField path="spring.jpa.hibernate.ddl-auto" type="string" default="update">
  Hibernate DDL mode for automatic schema management:
  
  - `update` - Update schema on startup (recommended for development)
  - `create` - Drop and recreate schema on startup
  - `create-drop` - Create on startup, drop on shutdown
  - `validate` - Validate schema without changes (recommended for production)
  - `none` - Disable automatic schema management
</ParamField>

<Warning>
  In production environments, consider using `validate` mode and managing schema changes through migration tools like Flyway or Liquibase.
</Warning>

### SQL Logging

```properties
spring.jpa.show-sql=false
spring.jpa.properties.hibernate.format_sql=true
```

<ParamField path="spring.jpa.show-sql" type="boolean" default="false">
  Enable SQL statement logging to console. Set to `true` for debugging queries in development.
</ParamField>

<ParamField path="spring.jpa.properties.hibernate.format_sql" type="boolean" default="true">
  Format SQL statements in logs for better readability when `show-sql` is enabled.
</ParamField>

### Database Initialization

```properties
spring.jpa.defer-datasource-initialization=false
spring.sql.init.mode=never
```

<ParamField path="spring.jpa.defer-datasource-initialization" type="boolean" default="false">
  Defer datasource initialization until after JPA entities are loaded. Set to `false` to prevent automatic data population.
</ParamField>

<ParamField path="spring.sql.init.mode" type="string" default="never">
  Control execution of SQL initialization scripts (`data.sql`, `schema.sql`):
  
  - `never` - Never execute initialization scripts (recommended for production)
  - `always` - Always execute scripts on startup
  - `embedded` - Execute only for embedded databases
</ParamField>

## H2 Console Disabled

The application explicitly disables H2 in-memory database features:

```properties
spring.h2.console.enabled=false
spring.autoconfigure.exclude=org.springframework.boot.autoconfigure.h2.H2ConsoleAutoConfiguration
spring.datasource.embedded-database-connection=none
```

This ensures only MySQL is used, preventing accidental use of embedded databases in production.

## Database Schema

### Core Tables

The application manages the following main entities:

#### Users and Authentication
- `user` - User accounts and profiles
- `role` - User roles (USER, ADMIN)
- `user_roles` - Many-to-many relationship between users and roles

#### Products and Catalog
- `producto` - Product catalog with details
- `categoria` - Product categories
- `review` - Product reviews and ratings

#### Shopping and Orders
- `carrito` - Shopping carts
- `carrito_item` - Items in shopping carts
- `compra` - Purchase orders
- `producto_comprado` - Products within purchases

#### Events and Messaging
- `event` - Outbound event log for published events
- `consumed_event_log` - Inbound event log from Kafka consumers with idempotency tracking

### Example Schema Creation

While Hibernate manages the schema automatically, here's the conceptual structure:

```sql
CREATE TABLE user (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    username VARCHAR(255) UNIQUE NOT NULL,
    email VARCHAR(255) UNIQUE NOT NULL,
    password VARCHAR(255) NOT NULL,
    first_name VARCHAR(255),
    last_name VARCHAR(255),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);

CREATE TABLE producto (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    codigo VARCHAR(50) UNIQUE NOT NULL,
    nombre VARCHAR(255) NOT NULL,
    descripcion TEXT,
    precio DECIMAL(10,2) NOT NULL,
    stock INT DEFAULT 0,
    categoria_id BIGINT,
    rate FLOAT DEFAULT 0.0,
    imagen_url VARCHAR(500),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (categoria_id) REFERENCES categoria(id)
);

CREATE TABLE compra (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    user_id BIGINT NOT NULL,
    estado VARCHAR(50) DEFAULT 'PENDING',
    total DECIMAL(10,2),
    fecha TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES user(id)
);

CREATE TABLE consumed_event_log (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    event_id VARCHAR(255) UNIQUE NOT NULL,
    event_type VARCHAR(255),
    origin_module VARCHAR(100),
    topic VARCHAR(100),
    partition_id INT,
    offset_value BIGINT,
    payload_json TEXT,
    status VARCHAR(50) DEFAULT 'PENDING',
    attempts INT DEFAULT 0,
    last_error TEXT,
    ack_sent BOOLEAN DEFAULT FALSE,
    ack_attempts INT DEFAULT 0,
    ack_last_at TIMESTAMP,
    ack_last_error TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_event_id (event_id),
    INDEX idx_status (status),
    INDEX idx_topic (topic)
);
```

## Connection Pooling

Spring Boot uses HikariCP as the default connection pool. Default settings are optimized for most use cases, but you can customize them:

```properties
# Optional: Advanced connection pool settings
spring.datasource.hikari.maximum-pool-size=10
spring.datasource.hikari.minimum-idle=5
spring.datasource.hikari.connection-timeout=30000
spring.datasource.hikari.idle-timeout=600000
spring.datasource.hikari.max-lifetime=1800000
```

### HikariCP Tuning

<ParamField path="spring.datasource.hikari.maximum-pool-size" type="integer" default="10">
  Maximum number of connections in the pool. Formula: `connections = ((core_count * 2) + effective_spindle_count)`
</ParamField>

<ParamField path="spring.datasource.hikari.minimum-idle" type="integer" default="5">
  Minimum number of idle connections maintained in the pool.
</ParamField>

<ParamField path="spring.datasource.hikari.connection-timeout" type="integer" default="30000">
  Maximum time (in milliseconds) to wait for a connection from the pool.
</ParamField>

## Production Recommendations

### Security

1. **Use SSL/TLS connections:**
   ```properties
   spring.datasource.url=jdbc:mysql://${SPRING_DATASOURCE_URL}?useSSL=true&requireSSL=true
   ```

2. **Create dedicated database user:**
   ```sql
   CREATE USER 'ecommerce_app'@'%' IDENTIFIED BY 'strong_password';
   GRANT SELECT, INSERT, UPDATE, DELETE ON ecommerce.* TO 'ecommerce_app'@'%';
   FLUSH PRIVILEGES;
   ```

3. **Restrict network access:**
   - Use VPC security groups to limit database access
   - Whitelist only application server IP addresses

### Performance

1. **Add database indexes:**
   ```sql
   CREATE INDEX idx_producto_codigo ON producto(codigo);
   CREATE INDEX idx_compra_user_fecha ON compra(user_id, fecha);
   CREATE INDEX idx_consumed_event_status ON consumed_event_log(status, updated_at);
   ```

2. **Enable query caching:**
   ```properties
   spring.jpa.properties.hibernate.cache.use_second_level_cache=true
   spring.jpa.properties.hibernate.cache.region.factory_class=org.hibernate.cache.jcache.JCacheRegionFactory
   ```

3. **Monitor slow queries:**
   ```sql
   -- Enable MySQL slow query log
   SET GLOBAL slow_query_log = 'ON';
   SET GLOBAL long_query_time = 2;
   ```

### High Availability

1. **Use Amazon RDS Multi-AZ** for automatic failover
2. **Configure read replicas** for read-heavy workloads:
   ```properties
   spring.datasource.hikari.read-only=true  # For read replica datasource
   ```
3. **Implement database backups** with automated snapshots

## Troubleshooting

### Connection Refused

```bash
com.mysql.cj.jdbc.exceptions.CommunicationsException: Communications link failure
```

**Solutions:**
- Verify MySQL is running: `systemctl status mysql`
- Check network connectivity: `telnet hostname 3306`
- Verify firewall rules allow port 3306
- Confirm `SPRING_DATASOURCE_URL` format is correct

### Authentication Failed

```bash
Access denied for user 'username'@'host'
```

**Solutions:**
- Verify credentials in environment variables
- Check user permissions: `SHOW GRANTS FOR 'username'@'%';`
- Ensure user can connect from application host

### Schema Validation Failed

```bash
org.hibernate.tool.schema.spi.SchemaManagementException: Schema-validation: missing table
```

**Solutions:**
- Set `spring.jpa.hibernate.ddl-auto=update` to auto-create tables
- Manually create database: `CREATE DATABASE ecommerce CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;`
- Run migration scripts if using Flyway/Liquibase

### Connection Pool Exhausted

```bash
HikariPool - Connection is not available
```

**Solutions:**
- Increase `maximum-pool-size`
- Check for connection leaks (unclosed connections)
- Reduce `connection-timeout` to fail faster
- Monitor active connections: `SHOW PROCESSLIST;`

## Database Migrations

For production deployments, consider using database migration tools instead of Hibernate's `ddl-auto`:

### Using Flyway

1. Add dependency:
   ```xml
   <dependency>
       <groupId>org.flywaydb</groupId>
       <artifactId>flyway-core</artifactId>
   </dependency>
   ```

2. Configure:
   ```properties
   spring.jpa.hibernate.ddl-auto=validate
   spring.flyway.enabled=true
   spring.flyway.baseline-on-migrate=true
   ```

3. Create migration scripts in `src/main/resources/db/migration/`:
   - `V1__Initial_schema.sql`
   - `V2__Add_consumed_event_log.sql`
