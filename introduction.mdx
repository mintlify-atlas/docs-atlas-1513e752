# Ecommerce Backend API

A comprehensive Spring Boot e-commerce backend service providing REST APIs for managing products, shopping carts, orders, user authentication, and event-driven integrations via Kafka.

## Architecture Overview

The Ecommerce Backend API is built with Spring Boot 3.5.4 and Java 17, leveraging:

- **Spring Data JPA** with MySQL for data persistence
- **Spring Security** for authentication and authorization
- **Spring Kafka** for event-driven messaging
- **JWT tokens** for stateless authentication
- **Keycloak** for OAuth2 integration
- **Docker** for containerization

<Note>
  The API follows a microservices architecture with dedicated modules for Sales (Ventas) and Inventory (Inventario) synchronization.
</Note>

## Technology Stack

- **Framework**: Spring Boot 3.5.4
- **Language**: Java 17
- **Database**: MySQL with JPA/Hibernate
- **Messaging**: Apache Kafka with SASL authentication
- **Security**: JWT, Keycloak OAuth2
- **Build Tool**: Maven 3.9+
- **Containerization**: Docker

## Core Capabilities

### User Management
- Registration with email verification
- JWT-based authentication
- Session management
- Password reset functionality
- Address management

### Product Catalog
- Product CRUD operations
- Category and brand management
- Product filtering and search
- Product reviews and ratings
- Favorite products

### Shopping Cart
- Add/remove cart items
- Update item quantities
- Cart persistence per user
- Cart checkout

### Order Management
- Create orders from cart
- Order history
- Order cancellation
- Purchase tracking

### Event Integration
- Kafka event consumers for inventory updates
- Event idempotency handling
- Retry mechanisms with exponential backoff
- Multi-module event synchronization

<Info>
  All endpoints are prefixed with `/api` and require authentication unless explicitly marked as public.
